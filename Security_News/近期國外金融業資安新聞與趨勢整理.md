## 近期國外金融業資安新聞與趨勢整理

**全球金融業資安威脅現況**

- 金融業長期以來一直是網路攻擊的重災區，僅次於醫療產業。2021年，全球有22%的勒索軟體攻擊針對金融業，且國家級攻擊者（如俄羅斯、北韓等）對金融機構的攻擊次數遠高於其他產業[^1_1][^1_3]。
- 金融服務業的數位轉型（如行動銀行、雲端服務、第三方供應鏈）帶來便利，也大幅增加了攻擊面，尤其第三方供應商成為資安防護的薄弱環節[^1_1][^1_4]。

**2025年主要資安威脅趨勢**

- **進階持續性威脅（APT）與AI驅動攻擊**：攻擊者利用AI與機器學習技術，進行更精準的目標分析與漏洞利用，並發動更難以偵測的魚叉式網釣與社交工程攻擊[^1_2][^1_3][^1_4]。
- **勒索軟體與資料外洩**：2024年勒索軟體如BianLian、Play、RansomHub等持續針對金融業，並預期2025年威脅將加劇。資料外洩事件頻傳，單一事件損失可達數百萬美元[^1_3][^1_4][^1_5]。
- **商業電子郵件入侵（BEC）**：BEC仍是最常見的攻擊手法，且有75%的事件能繞過多因素認證（MFA），顯示攻擊者手法日益精進[^1_6]。
- **供應鏈與第三方風險**：2023年美國某銀行因供應鏈攻擊導致130萬筆客戶資料外洩，凸顯第三方資安管理的重要性[^1_4]。
- **區塊鏈/加密金融攻擊**：2025年2月，去中心化金融平台zkLend因智能合約漏洞損失950萬美元，加密貨幣領域資安事件頻傳[^1_5]。

## 近期虛擬貨幣相關重大資安事件新聞

**Bybit 交易所史上最大駭客案（2025年2月）**

- 2025年2月21日，全球知名的加密貨幣交易所 Bybit 遭遇有史以來最大規模的駭客攻擊，駭客針對其以太坊多簽冷錢包，竊取高達約14.6~15億美元的加密資產，成為史上單一金額最高的虛擬貨幣竊案[^2_1][^2_3][^2_4][^2_5][^2_6][^2_8][^2_15]。
- 攻擊手法為滲透開發端，駭客取得 Bybit 內部 Safe 錢包開發人員的電腦控制權，並在前端 UI 植入惡意 JavaScript 代碼，誘導交易所人員在例行冷熱錢包轉帳時，簽署實際上將資金轉移至駭客地址的交易[^2_4][^2_6]。
- 被盜資產主要為以太幣（ETH）及質押以太幣，駭客隨後將資金分散轉移至多個錢包並利用去中心化交易所、跨鏈橋等方式進行洗錢[^2_3][^2_4][^2_6]。
- 此案經區塊鏈分析公司追蹤，顯示攻擊手法與北韓拉撒路集團（Lazarus Group）過往模式高度相似，並有跡象顯示與該集團有關聯[^2_5][^2_12][^2_13]。
- Bybit 創辦人Ben Zhou強調，僅部分冷錢包受影響，客戶資產1:1備付，交易所具備足夠儲備金應對損失[^2_4][^2_6][^2_15]。

**其他近期相關事件**

- 2025年2月24日，加密金融卡服務供應商 Infini 也遭駭，損失約4,950萬美元資金[^2_8][^2_11]。
- 2025年1月，整體加密貨幣產業因駭客攻擊損失約7,400萬美元，中心化交易所為重災區[^2_14]。
- 根據 Binance 及其他資安公司報告，2025年第一季加密貨幣產業因駭客攻擊、詐騙及漏洞利用，總損失超過20億美元，Bybit事件佔大宗[^2_12][^2_18]。

**產業反思與趨勢**

- Bybit 事件引發全球對交易平台冷錢包安全、內部管理、開發端資安防護的深刻反思[^2_4][^2_6][^2_10]。
- 事件凸顯即使冷錢包（離線錢包）也可能因開發端、前端簽名流程被滲透而失守，未來平台需加強多重簽名機制、內部人員資安訓練與供應鏈安全[^2_4][^2_6][^2_8]。
- 駭客攻擊手法持續進化，從釣魚、社交工程、私鑰管理漏洞到供應鏈攻擊，對整體加密產業安全防護提出更高要求[^2_8][^2_12][^2_17]。

**總結**

2025年2月的Bybit駭客案不僅刷新了單一事件的損失紀錄，也再次提醒業界，無論中心化或去中心化平台，資安防護都必須持續進化，否則將面臨巨額損失與市場信心危機[^2_1][^2_6][^2_8][^2_15]。

## 史上最大加密貨幣竊案：Bybit 交易所遭駭 15 億美元，揭露北韓駭客精密 UI 攻擊手法

**發布日期：** 2025年4月23日

**摘要：** 2025年2月21日，全球領先的加密貨幣交易所 Bybit 遭遇了加密貨幣歷史上最大規模的單一駭客攻擊事件。攻擊者利用精心策劃的多階段攻擊，成功滲透並操控了交易所用於管理以太坊（ETH）冷錢包的多重簽名（multisig）系統介面，最終竊取了價值近15億美元的加密資產，主要為ETH及質押ETH（stETH）。此次事件不僅震驚了全球加密貨幣社群，也再次突顯了交易所安全，特別是第三方供應鏈與前端介面安全的重要性。美國聯邦調查局（FBI）及多家網路安全公司事後將攻擊歸咎於與北韓政府有關聯的駭客組織「拉撒路集團」（Lazarus Group）。

---

**事件背景與規模**

2025年2月21日，星期五，加密貨幣市場經歷了黑暗的一天[^3_1]。總部位於杜拜、擁有超過6000萬用戶、日交易量達數百億美元的 Bybit 交易所，對外證實其系統遭到「複雜的網路攻擊」[^3_3][^3_6]。駭客成功繞過了交易所嚴密的多重簽名冷錢包安全機制，轉走了超過40萬枚ETH及數萬枚stETH、cmETH和mETH等相關資產，總價值約在14.6億至15億美元之間[^3_1][^3_2][^3_3][^3_5][^3_7]。

這起竊案的規模空前，遠超2022年 Ronin Network 的6.24億美元竊案，成為有記錄以來金額最高的加密貨幣交易所被駭事件[^3_1][^3_10][^3_11]。其損失金額甚至超過了北韓駭客在2024年全年竊取的加密貨幣總額（約13.4億美元）[^3_4][^3_10]。消息傳出後，市場反應劇烈，以太坊價格一度暴跌24%，比特幣也跌破9萬美元關口[^3_1]。

**駭客入侵軌跡與精密手法**

根據 Bybit 的內部調查以及多家網路安全公司（如 Sygnia、NCC Group、Chainalysis、TRM Labs 等）的分析，此次攻擊並非簡單的暴力破解或私鑰竊取，而是一場精心策劃、橫跨多個領域（macOS 惡意軟體、AWS 雲端入侵、應用程式安全、智能合約安全）的複雜供應鏈攻擊[^3_2][^3_8]。攻擊的完整時間線與手法大致如下：

1. **初期滲透（2月4日-5日）：** 攻擊的源頭最早可追溯至2月4日[^3_8]。駭客透過社交工程或釣魚攻擊，成功入侵了 Bybit 使用的第三方多簽錢包解決方案 Safe{Wallet}（前身為 Gnosis Safe）一名開發人員的 macOS 工作站[^3_1][^3_8][^3_9]。取得開發人員權限後，駭客在2月5日利用其 AWS 存取權杖（access token）進入了 Safe{Wallet} 的 AWS 雲端基礎設施[^3_1][^3_8]。
2. **潛伏與偵察（2月5日-17日）：** 駭客在 Safe{Wallet} 的 AWS 環境中潛伏了近兩週[^3_1][^3_8]。他們嘗試註冊自己的多因素認證（MFA）設備以維持訪問權限，但未能成功[^3_1]。隨後，駭客轉為低調偵察，分析 AWS 環境中的身份驗證機制和存取權杖的使用方式，並利用 VPN 隱藏蹤跡，劫持了開發人員用於訪問伺服器和提交程式碼的臨時會話權杖（session tokens）[^3_1]。
3. **惡意程式碼植入（2月17日-19日）：** 2月17日至19日期間，駭客將惡意 JavaScript 程式碼注入到 Safe{Wallet} 的使用者介面（UI）程式碼庫中，這些程式碼儲存在 AWS S3 儲存桶上[^3_1][^3_2][^3_8]。這段惡意程式碼被設計得極為隱蔽，包含特定的觸發條件，只在偵測到 Bybit 的特定冷錢包地址進行操作時才會啟動，對其他用戶則無影響[^3_1][^3_2][^3_8]。
4. **交易欺騙與執行（2月21日）：** 攻擊的高潮發生在2月21日。當天下午 UTC 時間 14:15 左右，駭客先執行了一筆 90 USDT 的測試交易，確認系統已被成功繞過[^3_7]。隨後，當 Bybit 的授權簽署人員（signers）透過 Safe{Wallet} 的網頁介面，準備執行一筆從冷錢包向熱錢包轉帳的例行內部資金轉移操作時，惡意 JavaScript 程式碼被觸發[^3_2][^3_4][^3_5][^3_6][^3_9][^3_12]。
    * **前端介面偽裝：** 被篡改的 UI 向簽署人員顯示了完全正常的交易細節，包括正確的目標熱錢包地址和合法的 Safe{Wallet} 網址[^3_1][^3_2][^3_5][^3_11][^3_12]。
    * **底層邏輯篡改：** 然而，在使用者看不到的底層，惡意程式碼修改了實際提交給硬體錢包（如 Ledger）簽署的交易內容[^3_9]。它利用了智能合約的 `delegatecall` 功能[^3_1][^3_8]。這個指令允許一個合約執行另一個合約的程式碼，但維持在原始合約的上下文中[^3_8]。駭客利用此特性，將一筆看似正常的轉帳請求，偷偷替換成一個指向預先部署好的惡意智能合約的委託調用[^3_5][^3_8]。
    * **簽署與控制權轉移：** Bybit 的簽署人員在被偽造介面誤導的情況下，未能仔細核對硬體錢包上顯示的真實交易細節，便批准並簽署了這筆交易[^3_9]。簽署完成後，惡意合約被執行，它立即更改了 Bybit 冷錢包智能合約的實施邏輯（implementation），引入了 `sweepETH` 和 `sweepERC20` 等惡意函數[^3_8]。這使得駭客無需再經過多重簽名批准，即可直接將冷錢包內的所有資金「掃除」到他們控制的地址[^3_8][^3_11]。
5. **資金盜取與清洗：** 從 UTC 14:16 開始，駭客迅速將冷錢包中的 401,347 ETH、90,375 stETH 及其他代幣轉出[^3_2][^3_7]。隨後在極短時間內（約82分鐘內），他們利用去中心化交易所（DEX）、跨鏈橋（cross-chain bridges）和無需 KYC 的即時兌換服務，將 stETH 等代幣兌換成 ETH，並將資金分散到大量中間錢包地址，試圖混淆追蹤（這種手法被稱為 chain hopping）[^3_1][^3_4][^3_7]。部分資金被轉換為比特幣（BTC）和 DAI 等其他加密貨幣[^3_4]。攻擊者還刻意讓一部分贓款在某些地址保持休眠狀態，企圖躲避風頭[^3_4]。UTC 15:51，Bybit 在 X 平台公開承認遭駭[^3_7]。攻擊完成兩分鐘後，駭客迅速從 Safe{Wallet} 介面中移除了惡意程式碼，以掩蓋蹤跡[^3_8]。

**歸因與後續**

事件發生後，多家區塊鏈分析公司如 Chainalysis、TRM Labs 和 Arkham Intelligence，以及網路安全公司如 Sygnia 和 Mandiant，透過鏈上資金流向分析和駭客 TTPs（戰術、技術和程序）比對，迅速將矛頭指向北韓的國家級駭客組織——拉撒路集團（Lazarus Group，也被 FBI 稱為 TraderTraitor、APT38、UNC4899、BlueNoroff）[^3_1][^3_4][^3_5][^3_6][^3_8][^3_9][^3_10]。證據包括：

* 攻擊手法（社交工程、供應鏈攻擊、複雜洗錢）與拉撒路過往的作案模式高度吻合[^3_4][^3_10]。
* 區塊鏈分析師 ZachXBT 透過懸賞計畫發現，部分被盜資金流入了曾用於2025年1月 Phemex 交易所 7000 萬美元竊案的地址，強烈暗示兩起事件為同一組織所為[^3_5][^3_8]。
* 2月26日，美國 FBI 正式發布公告，將此次攻擊歸咎於北韓駭客[^3_5][^3_7][^3_8][^3_9][^3_10]。

面對危機，Bybit 迅速採取行動：

* **用戶安撫與償付承諾：** CEO Ben Zhou 多次公開保證，只有單一冷錢包受影響，用戶資產安全且 1:1 儲備，即使無法追回全部損失，交易所也有超過200億美元的資產管理規模（AUM）和充足的償付能力覆蓋損失，甚至準備了過橋貸款以確保用戶提款順暢[^3_1][^3_3][^3_11]。
* **調查與追討：** Bybit 第一時間與執法機構（FBI、國際刑警組織等）以及多家頂級區塊鏈取證公司合作，追蹤資金流向[^3_5][^3_8]。該公司還設立了高達失竊金額10%（近1.4億美元）的賞金計畫，鼓勵白帽駭客和分析師協助追回資產[^3_5][^3_7]。這些合作取得了一定成效，例如 Tether 公司協助凍結了至少 18.1 萬與駭客地址關聯的 USDT[^3_5]，初期有超過4000萬美元的被盜資金被凍結或扣押[^3_5]。
* **安全強化：** Bybit 啟動了全面的內部安全審查和系統加固[^3_5]。同時，受到攻擊影響的 Safe{Wallet} 也進行了基礎設施重建、憑證輪換，並增加了額外的交易驗證措施[^3_5]。

**事件反思與產業影響**

Bybit 駭客案為整個加密貨幣產業敲響了警鐘：

* **冷錢包並非絕對安全：** 事件打破了「冷錢包絕對安全」的迷思，證明即使是離線存儲的多簽錢包，也可能因為其管理介面、第三方供應鏈或人為因素（如簽署流程疏忽）而被攻破[^3_1][^3_6][^3_8][^3_12]。
* **前端與 UI 安全至關重要：** 攻擊者並未直接破解智能合約或密碼學，而是利用了使用者介面的漏洞進行欺騙，顯示前端安全與使用者操作驗證（尤其是在硬體設備上核對）的重要性[^3_2][^3_5][^3_9][^3_12]。
* **供應鏈風險加劇：** 對 Safe{Wallet} 開發端的攻擊屬於典型的供應鏈攻擊，凸顯了加密企業對第三方服務商進行嚴格安全審核的必要性[^3_1][^3_5][^3_8]。
* **標準化與監管需求：** 與傳統金融相比，加密行業仍缺乏統一的安全標準和第三方風險評估框架，導致漏洞頻發[^3_8]。此次事件可能促使監管機構加強對交易所安全的審查[^3_1]。
* **產業韌性與合作：** 儘管損失巨大，Bybit 的快速反應、透明溝通和積極追討，以及產業各方的協作（如穩定幣發行商的凍結、分析公司的追蹤），也展現了加密生態系統在危機應對方面的韌性[^3_5]。

**結論**

Bybit 15億美元竊案是網路犯罪史上的一座黑暗里程碑，它清晰地展示了國家級駭客組織資源之雄厚、手法之精密。攻擊者利用對第三方軟體供應鏈的滲透和對使用者介面的巧妙操縱，成功繞過了複雜的多重簽名安全機制。此事件深刻地提醒所有加密貨幣參與者，從交易所到個人用戶，都必須持續加強安全意識、完善安全措施、警惕供應鏈風險，並在交易的每一個環節（尤其是在最終簽署確認時）保持最高的警惕性。唯有如此，才能在日益複雜的網路威脅環境中，保護好數位資產的安全。
