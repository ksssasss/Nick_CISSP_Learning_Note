# 2. 加密(encryption)技術要怎麼唸？請簡述加密的重點觀念

2.1 加密(encryption)技術要怎麼唸？請簡述加密的重點觀念
Answer：
探討元素及其目的
- 明文 Plain text
- 演算法 Cipher 
- 金鑰 Key
- 密文 Cipher text

重點觀念
- 雜湊、編碼 != 加密
- 對稱式：一把鑰匙
- 非對稱式：兩把鑰匙
- 數位簽章：被私鑰加密的雜湊值，完成身份比對與資料的真實性

```Wuson

• 其實這一題的重點是在強調唸書的方法。＂要唸那些東西？＂這樣的問題，其實也可以從”怎麼唸？＂去思考，比較能有結構的去申論重點。
• 加密技術要從【主要元素及其關係】的角度，把每個主要元素跟它們之間的關係弄清楚。
主要元素：明文 (要保護的機密資料)、金鑰(唯一的秘密)、加解密器(演算法/計算的過程)、密文(看不懂的明文)
加解密器的輸入：明文、金鑰
加解密器的輸出：密文
• 加解密器(cipher)要唸的主題：分類、比較、原理、特性及模式（限區塊型）。
1. 分類：
- 從金鑰來分類：使用一把鑰匙的對稱式 (symmetric)與使用二把鑰匙非對稱式(asymmetric)。
- 從資料來分類：對稱式還可分為區塊型(block cipher)及串流型(stream cipher)。
- 從數學來分類：對稱式使用布林代數；非對稱式可分為質因數分解、離散對數及楕圓曲線等。
2. 比較：請參閱講義(從金鑰長度、效能、用途等面向比較)
3. 原理：取代(substitution)與排列(permutation)；排列又稱為換位(transposition)。
4. 特性：混淆(confusion)與擴散(diffusion)。
5. 模式：區塊型加解密器的作業模式有ECB, CBC, OFB, CFB, CTR 。
• 金鑰(key)要唸的主題：從金鑰的生命週期的各個階段去唸，包含產生(Generation)、儲存(Storage)、建立(Establishment)、輪換(Rotation)、銷毀(Destruction)等。
```

# 3. 請說明【非對稱式加密】所採用之金鑰對(key pair)的技術特性及用途，並【比較】對稱及非對稱式加密技術的差異

3.1 請說明【非對稱式加密】所採用之金鑰對(key pair)的技術特性及用途
Answer：
金鑰對 key pair 特性：
- 兩把鑰匙同時出廠，精心配對
- 兩把鑰匙互為加解密
- 私鑰：給自己使用的金鑰，用於數位簽章
- 公鑰：給除了自己以外他人的金鑰

3.2 請【比較】對稱及非對稱式加密技術的差異
Answer：
1. 金鑰特性
	- 對稱式：一把鑰匙
	- 非對稱式：兩把鑰匙
2. 運算速度
	- 對稱式：效率高速度快
	- 非對稱式：超級無敵慢
3. 資料大小
	- 對稱式：沒限制
	- 非對稱式：運用於512位元以下(運算效率很差)
4. 應用情景
	- 對稱式：需要資料加密的情境廣泛利用
	- 非對稱式：金鑰交換＆數位簽章
5. 數學原理
	- 對稱式：布林值計算
	- 非對稱式：質因數分解、離散對數和橢圓曲線

# 7. 請比較編碼(encoding)、加密(encryption)及雜湊(hash)三者的差異

7.1 請比較編碼(encoding)、加密(encryption)及雜湊(hash)三者的差異
Answer：
首先核心觀念，雜湊、編碼 != 加密
- 編碼(encoding)目的：確保資料能被不同系統正確讀取與處理，目的在於資料格式的轉換
- 加密(encryption)目的：保護資料的機密性，防止未授權讀取（擁有金鑰才能解密）
- 雜湊(hash)目的：驗證資料的完整性，確保資料有無被修改，可以理解成fingerprint

```Wuson

以下補充:
• 編碼(encoding)、加密(encryption)及雜湊(hash)等演算法都會產出讓人看不懂、沒有意義的代碼/亂碼。編碼只是用一個符號或代號來替換另一個符號，其目的通常是為了資料的表示、儲存或傳輸；例如：ASCII, Unicode, BASE64。只要知道那一種編碼方式，任何人都可以進行解碼，還原原始資料。
• 加密則是為了保護機密資料。加密必須使用金鑰作為輸入參數，將明文轉換為密文；例如：DES, AES。只要掌握正確的金鑰，密文可以還原成明文（雙向函數）。
• 雜湊則是為了確保資料的完整性。雜湊的計算只需要輸入原始資料，不需要金鑰（加密才需要金鑰）；例如：MD5, SHA。雜湊結果長度相同，結果不可逆（單向函數，無法由雜湊值還原原始資料）。
• 加密不是編碼. 加密的目的是確保機密性, 編碼只是資料的表示方式. 加密一定要有金鑰, 編碼不需要金鑰.
• 雜湊在理論上反而沒有學者敢宣稱不會發生碰撞. 好的雜湊演算法只能說碰撞的機率很底, 但應該很少學者敢說機率為0或完全不會碰撞. 實務上, 好的雜湊演算法發生碰撞的機率很常被視為趨近於0而被忽略, 或視為不會碰鐘.

```

# 8. 請比較Nonce、鹽巴(salt)與初始向量(initialization vector)三者的差異

8.1 請比較Nonce、鹽巴(salt)與初始向量(initialization vector)三者的差異
Answer：

- Nonce：只用於一次性驗證的隨機性數值，經典案例如 OAuth
- 鹽巴(salt)：使用於「HASH」演算法，透過加入隨機的鹽值再進行雜湊，避免彩虹表＆預計算攻擊
- 初始向量(initialization vector)：使用於「加密」演算法，透過固定長度的隨機值，加入加密的過程來增加隨機性，主要的運用模板有 CBC, OFB, CFB, &CTR

```Wuson

以下補充:
• Nonce是number used once 的縮寫，也就是只用一次（不能重覆使用）的數值。雖然Nonce在實務上大多是隨機產生，但從字面意義來看，Nonce並沒有傳達出【隨機、可預測、不能外洩或只能用在加密】的概念。
• IV是一種隨機產生、只能用一次的Nonce。但IV (跟Nonce一樣) 不一定都要當作秘密(secret)加以保護。有人說Salt是一種Nonce，但Nonce是數值(number)而Salt是字串(string)，因此【本課程】不認為Salt是一種Nonce。
• 鹽巴(salt) 也是一種隨機產生的值(random value)，但Salt通常是以隨機字串(random string)的形式出現，並且搭配【雜湊】以產生新的使用者密碼，用來對付彩虹表(rainbow table)攻擊。此外，由於Hash也被用在金鑰產生及數位簽章（例如KDF ， key derivation function 與RSASSA-PSS RSA），因此Salt也會用在這些領域。
• 初始向量(initialization vector)是一種Nonce，通常應用在對稱式的區塊型加解密器，以增加破解的難度。向量是由一組純量(scalar)所組成。純量顧名思義是一個單一值，例如：10公里就是一個純量。但往西跑10公里除了距離, 也必須表達方向, 可以表示為 (西, 10)。如果用9點鐘方向來代表西，那就可以進一步寫成(9,10)的表示法。這種類似座標系(x,y,z,…)的表示法也被稱為tuple。因此，vector也常被稱為tuple. 在密碼學中, 一系列的0跟1也被視為一個向量。
• 計數器(Counter)是一種Nonce，但它不具備隨機性，因此不能稱為初始向量。

```
